GeoTools 9.4

Thank you for downloading GeoTools 9.4. This release was created on Jul 22, 2013.

This release contains only a subset of the current GeoTools code base. The full GeoTools code base contains additional unsupported modules covering experiments with a range of additional data formats. Only the modules which are considered to be stable have been included in this release.

For more information, and documentation on the GeoTools library please see the GeoTools User Guide.

The user guide includes the following tutorials to introduce you to GeoTools development:

  • Quickstart (Eclipse , Netbeans or Maven)

  • Feature Tutorial

  • Geometry CRS Tutorial

  • Query Tutorial

  • Image Tutorial

  • Map Style Tutorial

  • Additional tutorials cover topics such as making your own function, process or datastore.

Reference

Requirements

To use the GeoTools libraries you will need:

  • Java 6 or higher

GeoTools will benefit from the inclusion of Java Advanced Imaging (JAI) and the Java Image IO (JIO) extensions in your JRE. In addition some formats require the ImageIO-Ext additions which explicitly support raster formats such as MRSID and ECW. Detailed instructions on configuring your JDK with these optional extensions can be found in our users guide here.

Oracle (Optional)

In order to use the gt-oracle or gt-jdbc-oracle modules, you need the Oracle JDBC driver from Oracle. Unlike most external libraries used in GeoTools, we cannot redistribute this jar.

However, you can obtain them from the Oracle website, free of charge, after registering:

  • http://www.oracle.com/technetwork/database/features/jdbc/

Please see the users guide for detailed instructions.

Building (Optional)

For full details on requirements and build process please see the instructions on building (the following is provided as a quick reference only).

In order to build GeoTools you will need a copy of Maven. Maven is a project management tool from the Apache group.

At the time of writing, the build process was known to work with the Maven 3.0.5 release.

Once you have maven installed, you can perform a full build by moving to the GeoTools folder and typing:

mvn install

Please read the user guide for more information on using maven .

Getting Involved (Recommended)

The gt2-users mailing list is provided for general inquiries, with additional support options are available in the user guide.

The GeoTools project has a full Developers Guide documenting how you can take part in the project. If you are interested in the future development of GeoTools please join the geotools-devel mailing list and introduce yourself. We welcome contributions of new modules as well as keen developers who want to work on the project as a whole.

You can find out more about the mailing lists and recent news by going to the GeoTools Homepage.
